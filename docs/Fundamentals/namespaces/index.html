OOXML Namespaces

What Are XML Namespaces?â€‹

Namespaces prevent naming conflicts when combining XML from different specifications. Think of them as "last names" for XML elements.

Without namespaces:

<table>  <!-- HTML table? Database table? Word table? -->

With namespaces:

<w:tbl>  <!-- Word table -->
<html:table>  <!-- HTML table -->
<db:table>  <!-- Database table -->

Why OOXML Needs Namespacesâ€‹

Office Open XML combines multiple specifications:

  • WordprocessingML (Word documents)
  • DrawingML (Shapes, images, charts)
  • Relationships (Connecting parts)
  • Extended properties (Word 2010+ features)
  • Custom XML (Your data)

Each needs its own namespace to avoid conflicts.

Primary OOXML Namespacesâ€‹

Namespace Declarationsâ€‹

Where They Liveâ€‹

Namespaces are declared in the root element of each XML file:

<w:document 
    xmlns:w="http://schemas.openxmlformats.org/wordprocessingml/2006/main"
    xmlns:r="http://schemas.openxmlformats.org/officeDocument/2006/relationships"
    xmlns:wp="http://schemas.openxmlformats.org/drawingml/2006/wordprocessingDrawing"
    xmlns:a="http://schemas.openxmlformats.org/drawingml/2006/main"
    xmlns:pic="http://schemas.openxmlformats.org/drawingml/2006/picture"
    xmlns:v="urn:schemas-microsoft-com:vml"
    xmlns:w14="http://schemas.microsoft.com/office/word/2010/wordml">
  
  <w:body>
    <!-- Content using these namespaces -->
  </w:body>
</w:document>

Format: xmlns:prefix="namespace-uri"

  • prefix - Short name used in document (e.g., w, r)
  • namespace-uri - Full URI identifying the specification

Core Namespace: w: (WordprocessingML)â€‹

URI: http://schemas.openxmlformats.org/wordprocessingml/2006/main

Purpose: Main Word document elements

Elements:

  • w:document - Root
  • w:body - Body
  • w:p - Paragraph
  • w:r - Run
  • w:t - Text
  • w:tbl - Table
  • w:tr - Table row
  • w:tc - Table cell
  • w:pPr - Paragraph properties
  • w:rPr - Run properties
  • w:sdt - Content control

Usage:

<w:p>
  <w:r>
    <w:t>Hello World</w:t>
  </w:r>
</w:p>

ðŸ’¡ This is the most commonly used namespace in Word documents!

Relationships Namespace: r:â€‹

URI: http://schemas.openxmlformats.org/officeDocument/2006/relationships

Purpose: Reference relationships to other parts

Attributes:

  • r:id - Relationship ID
  • r:embed - Embedded resource reference
  • r:link - Linked resource reference

Usage:

<!-- Image reference -->
<a:blip r:embed="rId5"/>

<!-- Hyperlink -->
<w:hyperlink r:id="rId7">
  <w:r><w:t>Click here</w:t></w:r>
</w:hyperlink>

<!-- Header reference -->
<w:headerReference w:type="default" r:id="rId8"/>

ðŸ”— See Also: Relationship Files

WordprocessingDrawing Namespace: wp:â€‹

URI: http://schemas.openxmlformats.org/drawingml/2006/wordprocessingDrawing

Purpose: Image and shape positioning in Word documents

Elements:

  • wp:inline - Inline image (in text flow)
  • wp:anchor - Floating image (positioned independently)
  • wp:extent - Image dimensions
  • wp:docPr - Drawing properties

Usage:

<w:drawing>
  <wp:inline>  <!-- Image flows with text -->
    <wp:extent cx="914400" cy="914400"/>  <!-- 1 inch Ã— 1 inch -->
    <wp:docPr id="1" name="Picture 1"/>
    <!-- Image content -->
  </wp:inline>
</w:drawing>

DrawingML Namespace: a:â€‹

URI: http://schemas.openxmlformats.org/drawingml/2006/main

Purpose: Graphics, colors, effects, transforms

Elements:

  • a:graphic - Graphic container
  • a:graphicData - Graphic data
  • a:blip - Binary large image or picture (the actual image reference)
  • a:solidFill - Solid color fill
  • a:gradFill - Gradient fill
  • a:ln - Line
  • a:prstGeom - Preset geometry (shapes)

Usage:

<a:graphic>
  <a:graphicData uri="http://schemas.openxmlformats.org/drawingml/2006/picture">
    <pic:pic>
      <pic:blipFill>
        <a:blip r:embed="rId5"/>  <!-- Image reference -->
        <a:stretch>
          <a:fillRect/>
        </a:stretch>
      </pic:blipFill>
    </pic:pic>
  </a:graphicData>
</a:graphic>

Picture Namespace: pic:â€‹

URI: http://schemas.openxmlformats.org/drawingml/2006/picture

Purpose: Picture-specific elements

Elements:

  • pic:pic - Picture container
  • pic:blipFill - Image fill
  • pic:spPr - Shape properties
  • pic:nvPicPr - Non-visual picture properties

Usage:

<pic:pic>
  <pic:nvPicPr>
    <pic:cNvPr id="0" name="Property Photo"/>
    <pic:cNvPicPr/>
  </pic:nvPicPr>
  
  <pic:blipFill>
    <a:blip r:embed="rId5"/>
  </pic:blipFill>
  
  <pic:spPr>
    <a:xfrm>
      <a:ext cx="914400" cy="914400"/>
    </a:xfrm>
  </pic:spPr>
</pic:pic>

From AppraiSys: Property photos in appraisal reports use this structure.

VML Namespace: v:â€‹

URI: urn:schemas-microsoft-com:vml

Purpose: Vector Markup Language (legacy from Word 2003)

Status: Legacy but still used for backward compatibility

Elements:

  • v:shape - Vector shape
  • v:textbox - Text box
  • v:imagedata - Image data

Usage (less common now):

<v:shape style={{"width": "100pt", "height": "100pt"}}>
  <v:textbox>
    <w:txbxContent>
      <w:p><w:r><w:t>Text in box</w:t></w:r></w:p>
    </w:txbxContent>
  </v:textbox>
</v:shape>

ðŸ’¡ Modern approach: Use DrawingML (a:) instead when possible.

Extension Namespaces: w14:, w15:â€‹

URIs:

  • w14: - http://schemas.microsoft.com/office/word/2010/wordml
  • w15: - http://schemas.microsoft.com/office/word/2012/wordml

Purpose: Features added in Word 2010 and later

Examples:

  • w14:textFill - Text fill effects
  • w14:shadow - Shadow effects
  • w14:reflection - Reflection effects
  • w15:appearance - Appearance settings

Usage:

<w:rPr>
  <w14:textFill>
    <w14:gradFill>
      <!-- Gradient text fill (Word 2010+) -->
    </w14:gradFill>
  </w14:textFill>
</w:rPr>

âš ï¸ Compatibility: These features won't work in older Word versions.

Complete Example: Image with All Namespacesâ€‹

<w:document 
    xmlns:w="http://schemas.openxmlformats.org/wordprocessingml/2006/main"
    xmlns:r="http://schemas.openxmlformats.org/officeDocument/2006/relationships"
    xmlns:wp="http://schemas.openxmlformats.org/drawingml/2006/wordprocessingDrawing"
    xmlns:a="http://schemas.openxmlformats.org/drawingml/2006/main"
    xmlns:pic="http://schemas.openxmlformats.org/drawingml/2006/picture">
  
  <w:body>
    <w:p>  <!-- WordprocessingML: paragraph -->
      <w:r>  <!-- WordprocessingML: run -->
        
        <w:drawing>  <!-- WordprocessingML: drawing container -->
          
          <wp:inline>  <!-- WordprocessingDrawing: inline positioning -->
            <wp:extent cx="914400" cy="914400"/>
            <wp:docPr id="1" name="Photo"/>
            
            <a:graphic>  <!-- DrawingML: graphic -->
              <a:graphicData uri="...">
                
                <pic:pic>  <!-- Picture: picture element -->
                  <pic:nvPicPr>
                    <pic:cNvPr id="0" name="image.jpg"/>
                    <pic:cNvPicPr/>
                  </pic:nvPicPr>
                  
                  <pic:blipFill>
                    <a:blip r:embed="rId5"/>  <!-- Relationships: reference -->
                    <a:stretch>
                      <a:fillRect/>
                    </a:stretch>
                  </pic:blipFill>
                  
                  <pic:spPr>
                    <a:xfrm>
                      <a:ext cx="914400" cy="914400"/>
                    </a:xfrm>
                  </pic:spPr>
                </pic:pic>
                
              </a:graphicData>
            </a:graphic>
          </wp:inline>
        </w:drawing>
        
      </w:r>
    </w:p>
  </w:body>
</w:document>

Namespaces used:

  • w: - Document structure (document, body, p, r, drawing)
  • wp: - Image positioning (inline, extent, docPr)
  • a: - Graphics (graphic, graphicData, blip, xfrm, ext)
  • pic: - Picture specifics (pic, nvPicPr, blipFill, spPr)
  • r: - Relationship reference (embed)

Namespace Declaration Rulesâ€‹

Rule 1: Declare Before Useâ€‹

<!-- WRONG: namespace not declared -->
<w:document>
  <w:p><w:r><a:blip/></w:r></w:p>  <!-- a: not declared! -->
</w:document>

<!-- CORRECT: namespace declared -->
<w:document 
    xmlns:w="..."
    xmlns:a="...">
  <w:p><w:r><a:blip/></w:r></w:p>
</w:document>

Rule 2: Declare at Rootâ€‹

Best practice: declare all namespaces at the root element.

<!-- GOOD: All at root -->
<w:document xmlns:w="..." xmlns:r="..." xmlns:a="...">
  ...
</w:document>

<!-- WORKS but not ideal: Declare where used -->
<w:document xmlns:w="...">
  <w:drawing xmlns:a="...">
    <a:graphic>...</a:graphic>
  </w:drawing>
</w:document>

Rule 3: Prefix Can Vary (URI Matters)â€‹

The URI is what matters, not the prefix:

<!-- These are equivalent -->
<w:p xmlns:w="http://schemas.openxmlformats.org/wordprocessingml/2006/main">
  ...
</w:p>

<word:p xmlns:word="http://schemas.openxmlformats.org/wordprocessingml/2006/main">
  ...
</word:p>

However, stick to standard prefixes for clarity!

What Happens When Namespaces Are Wrong?â€‹

Missing Namespace Declarationâ€‹

<w:document>  <!-- w: declared -->
  <w:body>
    <a:graphic>  <!-- ERROR: a: not declared! -->
      ...
    </a:graphic>
  </w:body>
</w:document>

Result: Word can't parse the document. Shows "repair" dialog or fails to open.

Wrong Namespace URIâ€‹

<w:document xmlns:w="http://wrong-uri.com/word">
  <w:p><w:r><w:t>Text</w:t></w:r></w:p>
</w:document>

Result: Word doesn't recognize elements. Document may open but content is missing or malformed.

Missing Prefixâ€‹

<w:document xmlns:w="...">
  <body>  <!-- Should be w:body -->
    <p>   <!-- Should be w:p -->
      ...
    </p>
  </body>
</w:document>

Result: Elements not recognized. Document structure broken.

Debugging Namespace Issuesâ€‹

Using OpenXML SDK Productivity Toolâ€‹

  1. Open document in the tool
  2. Look at XML view
  3. Check root element declarations: 
    <w:document xmlns:w="..." xmlns:r="...">
  4. Verify all used prefixes are declared

Using Validatorâ€‹

using DocumentFormat.OpenXml.Validation;

var validator = new OpenXmlValidator();
var errors = validator.Validate(wordDoc);

foreach (var error in errors)
{
    if (error.Description.Contains("namespace"))
    {
        Console.WriteLine($"Namespace issue: {error.Description}");
    }
}

Manual Checkâ€‹

Extract DOCX and open XML files:

# Extract
Expand-Archive document.docx -DestinationPath extracted

# Check namespaces
Get-Content extracted/word/document.xml | Select-String "xmlns:"

Best Practicesâ€‹

âœ… DO:â€‹

  1. Declare all namespaces at root element
  2. Use standard prefix conventions (w:, r:, a:, etc.)
  3. Let OpenXML SDK handle namespaces when possible
  4. Validate documents after generation

âŒ DON'T:â€‹

  1. Forget to declare namespaces
  2. Use custom prefixes without good reason
  3. Mix different namespace declarations across parts
  4. Manually write namespace URIs (use SDK constants)

Test Your Understandingâ€‹

Question 1â€‹

What happens if you use <p> instead of <w:p> in document.xml?

Answer

Word won't recognize it as a paragraph because it's not in the WordprocessingML namespace. The document will likely fail to open or show a repair dialog. Always use the namespace prefix: <w:p>.

Question 2â€‹

Can you use a different prefix for the same namespace?

Answer

Technically yes, as long as the URI is correct:

<word:document xmlns:word="http://schemas.openxmlformats.org/wordprocessingml/2006/main">

However, this is confusing. Stick to standard prefixes (w:) for clarity.

Question 3â€‹

Where should you declare namespaces?

Answer

Best practice: Declare all namespaces in the root element of each XML file:

<w:document 
    xmlns:w="..."
    xmlns:r="..."
    xmlns:a="...">
  <!-- All namespaces declared here -->
</w:document>

This makes them available throughout the document.

ðŸ’¡ Remember: Namespaces are XML's way of preventing conflicts. Get them right, and elements work together harmoniously!
